WHAT IS "INDIA STACK"?

"India Stack" is PRware

A marketing term by iSPIRT branding separate closed systems as unified "open" infrastructure. Aadhaar is UIDAI. UPI is NPCI. ABDM is NHA. DigiLocker is MeitY. All closed source. All separately governed. The brand is unified. The code is proprietary.

Open-Washing Taxonomy

Four patterns of claiming "openness" without providing it:

01
API-as-Openness
Publishing endpoint documentation while keeping implementation proprietary. The interface is documented; the algorithm is secret. "You can call our system" ≠ "You can see our system."
→ Aadhaar publishes eKYC API specs but CIDR, biometric matching, and de-duplication remain proprietary. You cannot verify what happens after you call the API.
02
Spec-as-Source
Publishing a protocol specification while maintaining monopoly operation. The spec may not match what runs in production. "The standard is public" ≠ "The implementation is public."
→ UPI spec is public but: (1) only NPCI operates the network, (2) the production code is unknown, (3) you cannot verify the running system matches the spec.
03
Funnel Open-Sourcing
Open-sourcing wrappers (UI, SDKs) while keeping control components closed. The open parts are the sales funnel. The closed parts are where power resides. "The client is open" ≠ "The system is open."
→ MOSIP's registration workflows are open. Biometric matching (ABIS) is proprietary, you must buy from TECH5, Idemia, NEC. India exports this "open" framework while running closed-source Aadhaar at home.
04
DID-Washing
Using W3C vocabulary (DID, Verifiable Credentials, SSI, wallet) while anchoring all credentials to a centralized identity system. True W3C DID allows multiple issuers and verification without contacting the issuer. "Decentralized wallet" ≠ "Decentralized trust."
→ DigiYatra, Sunbird RC, Aadhaar Verifiable Credential all claim W3C DID/VC compliance but require Aadhaar as the sole identity anchor. The wallet is local; the trust is centralized.

Digital Public Good

AUDITABLE CODE ≠ PUBLIC SYSTEM
✗

Digital Public Infrastructure

Vague term. No formal definition. Can include closed-source. Self-declared.

UNVERIFIABLE

DPG is more verifiable than DPI, but observable code ≠ corrigible system. Without feedback channels (error correction, safe refusal, credible alternatives), a system cannot distinguish compliance from consent. Neither DPG nor DPI guarantees these structural properties.

Closed Control Loop

Exit is blocked technically, refusal is penalized legally, and correction is structurally unavailable.

India Stack lock-in is multi-dimensional:

Law
Enforces dependency. RTI-exempt. Litigation barred.
Technology
Enforces opacity. Closed source. No audit access.
Governance
Prevents correction. No constitutive input.
Exit
Punished by exclusion. No deletion mechanism.
Alternatives
Prohibited by regulation. Replication blocked.

None of these alone is decisive. Together, they form a closed control loop.

This is why the critique works cybernetically, not constitutionally.

Jurisdictional Arbitrage

India exports "open" digital frameworks while operating closed systems domestically.

THE PATTERN

Openness exists only at the boundary, not at the core.

This allows international promotion without domestic accountability.

What is marketed is architecture. What is withheld is control.

Other countries are sold a fictional openness. The defense, "they can choose how to implement it", ignores that India itself chose closure.

Commons Governance (Ostrom)

Elinor Ostrom (Nobel 2009) showed shared resources survive only when users are co-governors. India Stack fails her core principles:

Collective-choice arrangements?
No. NPCI/UIDAI not user-governed.
Monitoring by users?
No. RTI-exempt. No public audits.
Accessible conflict resolution?
No. Cannot sue UIDAI (Section 8).
Right to organize alternatives?
No. Legally and technically blocked.

A system used by everyone but governed by a few is not a commons, it is enclosure.

Why Not "Public"?

The test is not "who provides it" but whether citizens can observe, verify, and correct it. This is the cybernetic condition for democratic control.

Open Source Infrastructure
✓ Observable: Source code is public. Anyone can read it.
✓ Verifiable: Claims can be checked against implementation.
✓ Corrigible: Citizens can report bugs, submit fixes, fork if needed.
✓ Forkable: If governance fails, the code can be taken elsewhere.

Examples: Estonia's X-Road, UK's GOV.UK, USA's login.gov, all fully open source.

India Stack
✗ Not observable: Zero source code released. Black box.
✗ Not verifiable: Cannot check if implementation matches spec.
✗ Not corrigible: RTI-exempt. No public bug tracker. No citizen input.
✗ Not forkable: Alternatives legally/technically blocked.

17 years. 1.4 billion enrolled. Zero lines of code released for public audit.

A system you cannot observe is a system you cannot control. A system you cannot control is a system that controls you. → The Observability Problem

Error Asymmetry

In corrigible systems, error costs are shared and correctable. In India Stack, errors are externalized onto users.

Authentication failure?
User denied food, pension, services.
Data error?
User must prove identity again.
System failure?
Exclusion, not rollback.
Operator liability?
None. Correction is optional.

The system is fail-safe for the state. It is fail-deadly for users.

Errors are not bugs to be fixed. They are costs structurally externalized onto the governed.

Any system claiming to be public infrastructure must demonstrate corrigibility.
India Stack does not attempt to.

CLASSIFICATION

India Stack is a Club Good with Coercive Enrollment.
Not a Public Good. Not Open Source. Not a Commons.
