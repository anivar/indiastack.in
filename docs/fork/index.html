Testing India Stack against the basic requirement of open infrastructure: the ability to run your own instance.

What "Forkable" Means

Source code publicly available. Can be deployed independently. No permission required from original operators. This is the minimum bar for "open" infrastructure.

The Layers
System Forkable? Reality
Aadhaar No Closed source. Centralized by design. UIDAI operates the only instance. No federation protocol exists.
UPI No NPCI proprietary system. No public specification. Cannot build compatible implementation without NPCI approval.
DEPA / AA Partial Specification documents exist. But: implementations require ReBIT certification. Cannot operate without Sahamati approval.
DigiLocker No No self-hosting. Single government instance. Issuers must integrate with central system.
eSign No Requires licensed ASP (Application Service Provider). No open implementation. Tied to Aadhaar/eKYC.
ONDC No Private Section 8 company. Protocol not truly open, can you change the source code? Governance by private board.
ABDM / ABHA No Health ID tied to Aadhaar. Single operator (NHA). No alternative health identity system permitted.
CoWIN / U-WIN No Centralized vaccination registry. Now repurposed as U-WIN. No alternative immunization tracking allowed.
FASTag / NETC No NPCI-operated toll system. Mandatory for vehicles. No alternative electronic toll collection permitted.

What Truly Forkable Looks Like

For comparison, infrastructure that actually meets the "forkable" standard:

  • Estonia X-Road: Fully open source. Any country can deploy. Finland, Iceland, Japan use forks.
  • Gov.UK: Code on GitHub. Local councils can fork & adapt.
  • Matrix Protocol: Federated messaging. Anyone can run a server.
  • Email/SMTP: No single operator. Thousands of independent servers.

India Stack components meet none of these standards.

Code Is Law

When you cannot see the code, you cannot see the rules. When you cannot change the code, you cannot change the rules. This is governance by black box.

In closed systems, policy decisions are embedded in algorithms invisible to the public. Who gets authenticated? Who gets rejected? What data is collected? The code decides. And you cannot read it.

Why This Matters

When infrastructure cannot be forked, users have no exit. Operators face no competitive pressure. "Open API" becomes a facade over proprietary control.

True public digital infrastructure, like TCP/IP, HTTP, or email, can be implemented by anyone, anywhere, without permission. India Stack fails this test.

Verdict
Not Forkable. Not Open.
